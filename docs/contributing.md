# Contributing

We welcome contributions from students and instructors.

## How to contribute
- Fork the repo and create a branch for your work.
- Add or modify lab exercises under `web/content/` and update corresponding docs.
- If adding automation or images, update `docker/` or `workflow/` and include documentation in `docs/`.

## Standards
- Provide a README for any new lab or topology.
- Include topology diagrams, expected commands, and verification steps.
- Use clear commit messages and small, focused PRs.

## Code of Conduct
- Be respectful and constructive.
